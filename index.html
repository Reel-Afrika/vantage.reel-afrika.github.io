AI-Powered Financial
Analysis Platform

A multi-language system for automated market data aggregation, domain-grounded investment research, and structured analysis delivery — built with Python, Rust, and TypeScript in a native desktop shell.

Technology

Three-language architecture

Each language handles what it does best: Python for AI orchestration and data collection, Rust for high-throughput numerical computation, and TypeScript for a reactive UI — all connected through a shared DuckDB file and HTTP boundary.

Python AI Microservice

FastAPI server handling data ingestion from yfinance, SEC EDGAR XBRL API, FRED, and RSS feeds. RAG pipeline via LlamaIndex + ChromaDB. Thesis generation through the Anthropic Claude API. MCP server for chat-based access.

Rust Backend (Tauri)

Valuation engines (EPV, DCF, multiples, composite), quantitative screening pipelines, DuckDB read/write with file-level write-lock, tokio async runtime, and a cron scheduler for automated batch runs.

TypeScript Frontend

Frameworkless TypeScript bundled into Tauri's native WebView. uPlot for zero-dependency OHLCV charts. IPC via tauri::command — no separate server process, no Electron overhead (~5 MB install vs ~200 MB).

DuckDB (shared layer)

Single analytical database accessed by both Rust and Python. Columnar storage for screening across large ticker universes. File-level write lock prevents concurrent write conflicts. Serves as TTL cache, time-series store, and structured result archive.

RAG Knowledge Pipeline

Ingests domain documents (PDFs, markdown notes) into ChromaDB via LlamaIndex. Local embeddings with all-MiniLM-L6-v2 — no per-query API cost. Incremental re-ingestion via mtime manifest. Cited passages injected into Claude prompts.

Data Sources

Live market data (yfinance), structured multi-year financials via SEC EDGAR XBRL REST API (no key required), macro series from FRED via fredapi, and ticker-level news via Yahoo Finance RSS — all with configurable TTL caching.

Design

System architecture

The desktop app and Python microservice communicate over localhost HTTP. DuckDB is the single source of truth — Rust reads market data written by Python and writes screening results back for the frontend to display.

Engineering

Key highlights

Selected design decisions worth calling out.

🏗️

Schema-driven ORM (zero raw SQL)

Pydantic BaseModel subclasses generate all DDL and DML from field annotations — column names, types, nullability, defaults, and UNIQUE constraints. No hardcoded SQL strings outside the model layer.

🔒

File-level write lock

DuckDB is shared between Rust and Python. A .lock file held for each write transaction prevents simultaneous writes without blocking concurrent reads from either process.

🧠

Cited RAG → structured prompts

Domain PDFs and notes are chunked, embedded locally (no API cost), and stored in ChromaDB. 3–5 targeted queries per analysis inject cited passages directly into Claude prompts — grounding output in source material.

⚡

Rust valuation engines

EPV (Greenwald), DCF with sensitivity tables (Damodaran), peer multiples comparison, and composite bear/base/bull ranges — implemented as pure Rust with full intermediate value output for auditability.

📡

Multi-source data with TTL cache

Prices, fundamentals, macro series, SEC filings, and news each have independent TTL policies (4 h → 7 d). Cache misses fetch live; hits return instantly from DuckDB — no repeated API calls within a session.

🖥️

Native desktop via Tauri 2

Tauri wraps the TypeScript UI in the OS WebView — no bundled Chromium. macOS install is ~5 MB vs ~200 MB for an equivalent Electron app. IPC is type-safe via tauri::command with serde serialization.

🤖

MCP integration (Claude Code)

FastMCP server exposes platform capabilities as MCP tools — generate analysis, query the knowledge base, manage the watchlist, and trigger reports — all callable from Claude Code chat via stdio transport.

📅

Automated batch pipeline

tokio-cron-scheduler fires a weekly batch job: Rust scheduler → Python data fetch → Rust screening + valuation → Python thesis generation → structured markdown report written to the Obsidian vault.

📊

SEC EDGAR XBRL REST API

Structured multi-year financial statements pulled from data.sec.gov/api/xbrl/companyfacts/ — no third-party data subscription required. CIK lookup, 10-K/10-Q text extraction, and 7-day cache via DuckDB.

Code Quality

Engineering standards

The Python microservice is developed test-first with strict static analysis. CI runs on every push via GitHub Actions.

Every module has a corresponding test file. Fixtures are shared via conftest.py. External APIs (yfinance, FRED, feedparser, Anthropic) are mocked at the boundary — no network calls in CI. Pydantic models enforce types at runtime; mypy enforces them statically.
