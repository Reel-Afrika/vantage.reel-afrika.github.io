AI-Powered Financial
Analysis Platform

A multi-language system for automated market data aggregation, domain-grounded investment research, and structured analysis delivery — built with Python, Rust, and TypeScript in a native desktop shell.

Technology

Three-language architecture

Each language handles what it does best: Python for AI orchestration and data collection, Rust for high-throughput numerical computation, and TypeScript for a reactive UI — all connected through a shared DuckDB file and HTTP boundary.

Python AI Microservice

FastAPI server handling data ingestion from yfinance, SEC EDGAR XBRL API, and FRED. RAG pipeline via LlamaIndex + ChromaDB. Thesis generation and weekly report narrative via the Anthropic Claude API. MCP server with 21 tools for chat-based access.

Rust Backend (Tauri)

Valuation engines (EPV, multiples, composite), quantitative screening pipelines, DuckDB read/write with file-level write-lock, tokio async runtime, and a cron scheduler for automated batch runs.

TypeScript Frontend

Frameworkless TypeScript bundled into Tauri's native WebView. uPlot for zero-dependency OHLCV and valuation-multiples charts. Accordion candidate panels with radar charts. IPC via tauri::command — no separate server process, no Electron overhead (~5 MB vs ~200 MB).

DuckDB (shared layer)

Single analytical database accessed by both Rust and Python. Columnar storage for screening across the full S&P 500 universe. File-level write lock prevents concurrent write conflicts. Serves as TTL cache, time-series store, and structured result archive.

RAG Knowledge Pipeline

Ingests domain documents (PDFs, markdown notes) into ChromaDB via LlamaIndex. Local embeddings with all-MiniLM-L6-v2 — no per-query API cost. Incremental re-ingestion via mtime manifest. Cited passages injected into Claude prompts.

Static Chart Generation

matplotlib produces three PNGs per thesis at zero token cost: 5-year revenue & earnings bar chart (EDGAR XBRL), 1-year price vs 200d MA line chart (yfinance), and valuation multiples vs 5-year historical average. Images are embedded directly in Obsidian markdown.

Data Sources

Live market data (yfinance), structured multi-year financials via SEC EDGAR XBRL REST API (no key required), macro series from FRED via fredapi — all with configurable TTL caching in DuckDB.

Screening

Three-tier pipeline

Every scheduled run works through three stages in sequence. The universe starts from the current S&P 500 constituent list (~500 tickers), refreshed monthly. Each stage narrows the field and adds richer AI-generated context.

Engineering

Key highlights

Selected design decisions worth calling out.

🏗️

Schema-driven ORM (zero raw SQL)

Pydantic BaseModel subclasses generate all DDL and DML from field annotations — column names, types, nullability, defaults, and UNIQUE constraints. No hardcoded SQL strings outside the model layer.

🔒

File-level write lock

DuckDB is shared between Rust and Python. A .lock file held for each write transaction prevents simultaneous writes without blocking concurrent reads from either process.

🧠

Cited RAG → structured prompts

Domain PDFs and notes are chunked, embedded locally (no API cost), and stored in ChromaDB. 3–5 targeted queries per analysis inject cited passages directly into Claude prompts — grounding output in source material.

⚡

Rust valuation engines

EPV (Greenwald) and peer multiples comparison implemented as pure Rust with full intermediate value output for auditability. Sector median P/E, EV/EBITDA, P/FCF, and EV/S computed per ticker for premium/discount positioning.

📡

Multi-source data with TTL cache

Prices, fundamentals, macro series, and SEC filings each have independent TTL policies (4 h → 7 d). Cache misses fetch live; hits return instantly from DuckDB — no repeated API calls within a session.

🖥️

Native desktop via Tauri 2

Tauri wraps the TypeScript UI in the OS WebView — no bundled Chromium. macOS install is ~5 MB vs ~200 MB for an equivalent Electron app. IPC is type-safe via tauri::command with serde serialization.

🤖

MCP integration (21 tools)

FastMCP server exposes 21 platform tools — generate analysis, manage the universe, refine screening criteria via a plain-English questionnaire, query the knowledge base, manage the watchlist, and trigger reports — callable from Claude Code chat via stdio transport.

📅

Automated batch pipeline

tokio-cron-scheduler fires a weekly batch: Rust scheduler → Python data fetch → Rust screening + valuation → Python candidate promotion → thesis generation with embedded matplotlib charts → structured markdown report written to the Obsidian vault.

📊

SEC EDGAR XBRL REST API

Structured multi-year financial statements pulled from data.sec.gov/api/xbrl/companyfacts/ — no third-party data subscription required. CIK lookup, 10-K/10-Q text extraction, and 7-day cache via DuckDB.

🌐

Dynamic S&P 500 universe

The screening universe is the current S&P 500 constituent list fetched from Wikipedia and upserted on demand — never a hardcoded ticker list. Each company carries its GICS sector for colour-coded badges and filter controls across the UI.

⚠️

Thesis & watchlist freshness

Theses older than 14 days are flagged stale (orange → red badge) in the Tauri thesis viewer. Watchlist entries are re-scored each pipeline run; any that fall below the qualifying threshold show a "no longer qualifies" banner with a one-click remove.

🎛️

Guided criteria refinement

MCP tools refine_growth_criteria and refine_value_criteria ask 5 plain-English questions about risk tolerance and investment preferences, then translate answers into updated scoring rule weights — no JSON or finance jargon required.

Code Quality

Engineering standards

Both the Python microservice and TypeScript frontend are developed test-first with strict static analysis. CI runs on every push via GitHub Actions.

Every module has a corresponding test file. Fixtures are shared via conftest.py. External APIs (yfinance, FRED, Anthropic) are mocked at the boundary — no network calls in CI. Pydantic models enforce types at runtime; mypy enforces them statically. TypeScript is checked with tsc --noEmit and tested with Vitest (jsdom environment).
